  <name>Task 1: Create BeforeAfterSlider component</name>
  <files>src/components/ui/BeforeAfterSlider.tsx</files>
  <action>
Create a draggable image comparison slider using native Pointer Events for unified mouse+touch handling.

**BeforeAfterSlider.tsx** — 'use client' component:
- Import { useCallback, useRef, useState } from 'react'
- Import Image from 'next/image'
- Import { cn } from '@/lib/utils'

**Props interface (BeforeAfterSliderProps):**
- beforeSrc: string
- afterSrc: string
- beforeAlt: string
- afterAlt: string
- className?: string

**State:**
- position: number (0-100), default 50 — represents the divider position as a percentage

**Pointer event handlers:**
- handlePointerDown: calls (e.target as HTMLElement).setPointerCapture(e.pointerId) — this ensures drag continues even if pointer leaves the container
- handlePointerMove: only fires when button is pressed (e.buttons > 0 || e.pressure > 0). Calculates x position as percentage from container rect: ((e.clientX - rect.left) / rect.width) * 100, clamped to 0-100

**Render structure:**
1. Container div:
   - ref={containerRef}
   - className: cn('relative aspect-[16/10] select-none overflow-hidden touch-none', className)
   - aspect-[16/10] prevents CLS from unsized images (Research Pitfall 7)
   - touch-none prevents mobile browser from intercepting touch for page scroll (Research Pitfall 5)
   - Pointer event handlers on the container (not the handle)
   - ARIA: role="slider", aria-label="Before and after comparison", aria-valuenow={Math.round(position)}, aria-valuemin={0}, aria-valuemax={100}, tabIndex={0}
   - Keyboard handler: ArrowLeft decreases by 5, ArrowRight increases by 5

2. "Before" image — full width, always visible:
   - Next.js Image with fill, className="object-cover", sizes="(max-width: 768px) 100vw, 50vw"

3. "After" image — clipped from right based on position:
   - Absolutely positioned div with clipPath: inset(0 0 0 ${position}%)
   - Next.js Image with fill inside the clip div

4. Drag handle — vertical gold line at position:
   - Absolutely positioned div, w-0.5, bg-gold, full height, left: ${position}%
   - Circular knob at vertical center: w-10 h-10 rounded-full, border-2 border-gold, bg-void/80
   - Unicode arrows inside knob (aria-hidden) for visual affordance

Do NOT use Motion's drag prop — it requires domMax which is not loaded (Research Pitfall 1). Use native Pointer Events only.

  <name>Task 2: Create ProcessTimeline and ScrollAssembly components</name>
  <files>src/components/ui/ProcessTimeline.tsx, src/components/ui/ScrollAssembly.tsx</files>
  <action>
Create two scroll-driven animation components that use Motion's useScroll + useTransform hooks.

**ProcessTimeline.tsx** — 'use client' component:
- Import { m, useScroll, useTransform } from 'motion/react'
- Import { useRef } from 'react'

- TimelineStep interface: { number: string, title: string, description: string }
- ProcessTimelineProps: { steps: TimelineStep[], className?: string }

- Create containerRef and useScroll with target: containerRef, offset: ['start end', 'end center'] — starts drawing when container bottom enters viewport, finishes when container center hits viewport center
- Map scrollYProgress to pathLength: useTransform(scrollYProgress, [0, 1], [0, 1])

- Render structure:
  1. Container div with ref
  2. Relative wrapper div inside
  3. SVG connecting line (absolute, left-6, top-0, h-full, w-1):
     - viewBox="0 0 2 100", preserveAspectRatio="none"
     - Background track: line element with stroke="var(--color-charcoal)", strokeWidth="2"
     - Animated gold line: m.line with stroke="var(--color-gold)", strokeWidth="2", style={{ pathLength }}, strokeDasharray="0 1" (CRITICAL: prevents SSR flash — Research Pitfall 3)
     - aria-hidden="true" on the SVG (decorative)
  4. Steps mapped with m.div:
     - Each step: relative pl-16 pb-16 last:pb-0
     - whileInView with staggered delay (i * 0.15), once: true, luxury easing
     - initial={{ opacity: 0, y: 30 }}, whileInView={{ opacity: 1, y: 0 }}
     - Step dot on line: absolute left-4, w-5 h-5 rounded-full border-2 border-gold bg-void
     - Large step number behind text: font-display text-6xl text-gold/10 (low opacity gold)
     - Title: font-display text-xl text-cream with -mt-8 to overlap number
     - Description: mt-2 text-warm-gray

**ScrollAssembly.tsx** — 'use client' component:
- Import { m, useScroll, useTransform } from 'motion/react'
- Import { useRef } from 'react'

- AssemblyPiece interface: { id: string, label: string, initialX: number, initialY: number, initialRotate: number }
- ScrollAssemblyProps: { pieces: AssemblyPiece[], className?: string }

- This component takes an array of "pieces" (abstract shapes) that start scattered and converge to a centered assembled position as the user scrolls.

- Create containerRef and useScroll with target: containerRef, offset: ['start end', 'end center']
- For each piece, create motion values:
  - x: useTransform(scrollYProgress, [0, 0.8], [piece.initialX, 0]) — converges to center
  - y: useTransform(scrollYProgress, [0, 0.8], [piece.initialY, 0]) — converges to center
  - rotate: useTransform(scrollYProgress, [0, 0.8], [piece.initialRotate, 0]) — straightens
  - opacity: useTransform(scrollYProgress, [0, 0.2], [0.3, 1]) — fades in from dim

- Render structure:
  1. Tall container div with ref, min-h-[150vh] (needs scroll distance for animation to play out)
  2. Sticky inner wrapper: sticky top-0 h-screen flex items-center justify-center — keeps the assembly centered in viewport while scrolling
  3. Relative assembly area (w-80 h-80 or similar) containing:
     - m.div for each piece with style={{ x, y, rotate, opacity }}
     - Each piece renders as a decorative placeholder: a colored rectangle/shape with the piece label
     - Use bg-charcoal border border-gold/30 with different sizes to represent metal parts

- NOTE: The pieces array with specific initialX/Y/rotate values will be configured by the test harness and later by the Phase 4 portfolio section. The component itself is generic — it animates any set of pieces from scattered to assembled.

Both files must use m.* (not motion.*) per LazyMotion strict mode.
