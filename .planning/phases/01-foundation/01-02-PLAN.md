Create `src/providers/Providers.tsx` as a Client Component wrapping LazyMotion and MotionConfig:

```typescript
// src/providers/Providers.tsx
'use client'

import { LazyMotion, domAnimation, MotionConfig } from 'motion/react'

interface ProvidersProps {
  children: React.ReactNode
}

export function Providers({ children }: ProvidersProps) {
  return (
    <LazyMotion features={domAnimation} strict>
      <MotionConfig reducedMotion="user">
        {children}
      </MotionConfig>
    </LazyMotion>
  )
}
```

CRITICAL details per research:
- Import from `"motion/react"` — NOT `"motion/react-m"` (known bug #3091)
- Use named export `{ m }` from `"motion/react"` in all future components — NOT `motion.div`
- `strict` prop on LazyMotion throws build errors if any `motion.*` component is imported (this is the guard rail)
- `reducedMotion="user"` on MotionConfig auto-disables transform/layout animations while preserving opacity

Create `src/components/layout/GrainOverlay.tsx` as a Server Component (no 'use client' needed):

```typescript
// src/components/layout/GrainOverlay.tsx
export function GrainOverlay() {
  return (
    <div
      className="pointer-events-none fixed inset-0 z-50 opacity-[0.04] motion-reduce:hidden"
      style={{
        backgroundImage: 'url(/textures/grain.png)',
        backgroundRepeat: 'repeat',
        mixBlendMode: 'overlay',
      }}
      aria-hidden="true"
    />
  )
}
```

CRITICAL details per research:
- `pointer-events-none` is MANDATORY — without it, the overlay intercepts all clicks (Pitfall 5)
- `aria-hidden="true"` for screen reader accessibility
- `motion-reduce:hidden` hides grain when user has prefers-reduced-motion enabled
- `z-50` ensures overlay sits on top of all content
- `opacity-[0.04]` gives 4% opacity (within the 3-5% range from DSGN-03)
- `mix-blend-mode: overlay` blends naturally with the dark background

Generate the grain texture PNG at `public/textures/grain.png`:
- Create a ~200x200px tileable noise texture programmatically using a canvas script or Node.js script
- The texture should be transparent background with white noise dots — white noise at low opacity with overlay blend mode creates the grain effect on dark backgrounds
- If programmatic generation is complex, create a minimal placeholder: a simple 200x200 transparent PNG with random white dots at ~20% density
- The file should be small (under 20kb) and tileable
