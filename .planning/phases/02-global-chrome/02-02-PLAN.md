<task type="auto">
  <name>Task 1: Create CustomCursor component with rAF + lerp tracking</name>
  <files>src/components/layout/CustomCursor.tsx</files>
  <action>
Create `src/components/layout/CustomCursor.tsx` as a `'use client'` component.

The cursor MUST use raw `requestAnimationFrame` + linear interpolation (lerp) — NOT Motion's `animate` prop or React state for position. This is the critical performance requirement (Pitfall 2 from research).

**Implementation:**
1. Define a `lerp(a: number, b: number, t: number): number` helper that returns `a + (b - a) * t`.
2. Use `useRef` for: `cursorRef` (HTMLDivElement), `posRef` (current {x, y}), `targetRef` (mouse {x, y}), `rafRef` (animation frame ID), `isHoveringRef` (boolean).
3. Create a `useCallback` animate function that:
   - Lerps `posRef.current` toward `targetRef.current` using `CURSOR_LERP_SPEED` (import from `@/lib/constants`)
   - Writes `transform: translate(${x}px, ${y}px)` directly to `cursorRef.current.style.transform` — bypasses React render cycle
   - Schedules next frame via `requestAnimationFrame(animate)`
4. In a `useEffect`:
   - Add `mousemove` listener on `window` that updates `targetRef.current` with `e.clientX, e.clientY`
   - Add `mouseover` listener on `document` using event delegation: check if `(e.target as HTMLElement).closest('a, button, [data-cursor="pointer"], [data-cursor="view"]')` matches. If yes, add `cursor-expanded` class to `cursorRef.current`.
   - Add `mouseout` listener on `document` with same `closest()` check. If match, remove `cursor-expanded` class.
   - Start rAF loop: `rafRef.current = requestAnimationFrame(animate)`
   - Cleanup: remove all listeners, `cancelAnimationFrame(rafRef.current)`
5. Render a single `<div>` with:
   - `ref={cursorRef}`
   - `className="custom-cursor pointer-events-none fixed top-0 left-0 z-[9999]"`
   - `aria-hidden="true"`

**data-cursor attribute system:** The `closest()` selector already supports `[data-cursor="view"]` for future Phase 4 portfolio image hover. No additional work needed now — portfolio images will simply add `data-cursor="view"` when they exist.

IMPORTANT: All browser API access (window, document, requestAnimationFrame) MUST be inside `useEffect` — never during render. This prevents hydration mismatch (Pitfall 8 from research).
  </action>

    will-change: transform;
  }

    height: 40px;

    margin-left: -20px;

  }
